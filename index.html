Git

A platform is required to keep changes git

Git is a DevOps tool used for source code management. It is a free and open-source version control system used to handle small to very large projects efficiently. Git is used to tracking changes in the source code, enabling multiple developers to work together on non-linear development. Linus Torvalds created Git in 2005 for the development of the Linux kernel.

Git is easy to learn and has a tiny footprint with lightning fast performance. It outclasses SCM tools like Subversion, CVS, Perforce, and ClearCase with features like cheap local branching, convenient staging areas, and multiple workflows.

Lets Start!!!

  • git init

    git init is used to create local repository in your IDE You can create an empty repositiry eith it. You can also turn an ordinary projects into magical repositories Always the first command in the journey of your github.

    This will create a folder named .git inside your project folder.It contains all of the necessary Git metadata for the new repository.In some systems it can be hidden You can make it visisble from view option.


  • git config

    The git config command is a convenience function that is used to set Git configuration values on a global or local project level. It will modify the configuration file. You can add user email or your user name to the git configuration.


  • git add

    The git add command adds a change in the working directory to the staging area.It helps in maintaing versions. "git add . command "add all the files in one go to staging area. You can also add a particular file by "git add filename".

  • git log

    git log shows the history of all the changes made in master branch or specified branch

  • git commit

    To move from staging to commit for our repositiry Adding commits keep track of our progress and changes as we work. Git considers each commit change point or "save point". You can track your progress and go back to previous version if you want to in case of a bug. commit command require a message with it to keeep track of changes in readable forms. To add message -m option is used.


  • git remote add origin

    To move data from your local repositiry to the remote repositories u can add link of that repositiry to your project using git remote command.


  • git clone

    git clone is a Git command line utility which is used to target an existing repository and create a clone, or copy of the target repository. The original repository can be located on the local filesystem or on remote machine accessible supported protocols. The git clone command copies an existing Git repository. Here the shaded area contains the https link of your remote repository. you can copy the address from github website.


  • git push

    The git push command is used to upload local repository content to a remote repository. Pushing is how you transfer commits from your local repository to a remote repository.After a local repository has been modified a push is executed to share the modifications with remote team members. Here git push add all the changes to remote repository.
